# Contributing

Thanks for your interest in Cerebro Web.

## Pull Requests

- Keep changes focused and include tests or validation steps when applicable.
- Run `npm run lint`, `npm run build`, `npm run oss:audit`, and `npm run audit:high` before opening a PR.
- Do not include customer data, credentials, internal hostnames, private repository names, account IDs, or environment-specific deployment details.

## Security Issues

Report security vulnerabilities privately to security@writer.com instead of opening a public issue.

## Support

This project is maintained by WRITER on a best-effort basis. Functional bugs can be filed through GitHub Issues; feature requests and PRs may be declined if they do not align with the project roadmap.
